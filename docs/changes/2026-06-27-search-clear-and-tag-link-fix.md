# 検索クリアボタン + viewer 内タグ/作者リンクの修復

## 背景

1. ビューワー内のメタパネルにあるタグ/作者の chip をタップしても、 期待されている
   「一覧画面に遷移してそのタグ/作者で絞り込まれた状態で表示」 が動かない。
   実際には `/index.html` には飛ぶが **クエリ ?q=... が剥落** している。
2. 一覧画面の検索バーに入力した文字をクリアするのが手間 (全選択して削除など)。
   PWA や一部 iOS Safari ではブラウザ標準の `<input type="search">` のクリア
   ボタンが表示されないことがあり、 ユーザーから不便との指摘。

## 現状の挙動

### 1. タグ/作者リンクの剥落

- ビューワー: メニューを開く → 「タグ」 行の chip をタップ → URL は
  `/index.html` に飛ぶが、 `?q=タグ名` が **消えている**
- 原因: `web/server.ts` の `app.get("/", (c) => c.redirect("/index.html"))`
  が **元 URL のクエリ文字列を保持していない**。 タグ chip は `/?q=shayo`
  をクリックするが、 サーバが 302 で `/index.html` に redirect する際に
  `?q=shayo` を捨てる
- 結果: 一覧画面が q なしで開く → 絞り込まれない

### 2. 検索バーのクリア

- 一覧画面の topbar に `<input type="search" id="search">` がある
- iOS Safari / Chromium の一部では入力中も × アイコンが小さくしか出ない or
  出ない
- 入力をやめたい時、 全選択 → Delete などのキー操作が必要
- (空状態画面の「条件をクリア」 ボタンは 0 件時にしか出ず、 通常入力中には使えない)

## 変更後の挙動

### 1. タグ/作者リンクが正しく絞り込みへ飛ぶ

- ビューワー内のタグ/作者/シリーズ chip をタップ → **一覧画面が開き、
  検索バーにその語が入った状態で結果が絞り込まれている**
- 一覧画面のカード下の作者リンクも同様 (こちらは既に動作している)
- バグ修正: `GET /` の redirect で元 URL の query を保持

### 2. 検索バーのクリアボタン

- 検索 input の **右端内側に × アイコン** が出る
- **入力に文字があるときだけ** 表示。 空になると消える
- × を押すと:
  - 検索クエリ (`state.query`) のみクリア。 ディレクトリ・ステータスフィルタは保持
  - 検索バーが空になる
  - URL から `?q=` が外れる
  - 一覧と sections が再ロードされる (検索クリア = 全件 + sections 復活)
  - フォーカスは検索 input に残す (続けて別の語を入力できるように)
- スマートフォン (タッチ) でも 32×32 程度の十分な押下面積を確保

## スコープ外

- 検索履歴・サジェスト機能
- 検索クエリの保存・お気に入り
- ディレクトリ/ステータスフィルタの個別クリアボタン (= 既存「条件をクリア」 で 一括クリア可能)
- 空状態の「条件をクリア」 ボタンの統廃合 (今回は触らない)

## E2E テストチェックリスト

### 正常系

- [ ] ビューワーで タグ chip をタップ → 一覧画面 (`/index.html` または `/`) に遷移
- [ ] 遷移後の URL に `?q=<タグ名>` が **含まれる**
- [ ] 遷移後の検索 input にタグ名が入っている
- [ ] 遷移後のグリッドがそのタグで絞り込まれている (該当書籍のみ表示)
- [ ] ビューワーの作者 chip タップでも同様 (`?q=<作者名>`)
- [ ] ビューワーのシリーズ chip タップでも同様
- [ ] 一覧画面で検索 input に何か入力すると **右端に × アイコンが現れる**
- [ ] × をタップすると検索バーが空になり、 URL から `?q=` が消える
- [ ] × をタップすると現在の directory / status フィルタは **保持される**
- [ ] × をタップしたあと検索バーがフォーカス状態を保つ

### 異常系

- [ ] 検索バーが空のときは × アイコンが表示されない
- [ ] サーバが落ちている時にタグ chip をタップしても、 アプリ自体は壊れない
      (リダイレクト失敗時はブラウザのエラーページ、 復旧で正常動作)

### エッジケース

- [ ] 検索バーに **空白だけ** 入力されたとき × が出る? (= 文字があれば true で扱う)
- [ ] タグ名にエンコード必要な文字 (`&`, スペース等) が含まれても、 URL エンコードされて正しく遷移
- [ ] タグ名が日本語 (例: 「忍者」) でも遷移先で正しくデコードされて表示
- [ ] × をタップした直後にすぐ別の語を入力できる (debounce 干渉なし)
- [ ] ディレクトリ絞り込み中 + q あり の状態で × → q だけ消えて directory は保持
- [ ] スマートフォン幅 (402px) で × アイコンが押せる (input の placeholder と被らない)
- [ ] viewer の PWA (standalone) からタグタップで戻る場合も q が保持される
- [ ] viewer のメニュー (`#menu-overlay`) を閉じる前にタップしても遷移が動く
