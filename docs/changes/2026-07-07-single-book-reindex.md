# 特定の書籍のみを再インデックスする機能

## 背景

現状、書籍のインデックスをリフレッシュする手段は「差分再インデックス」と「全件再インデックス」の 2 種類のみで、いずれも設定画面から全書籍を対象に走らせる。

以下のケースで「この 1 冊だけ入れ直したい」というニーズがある:

- zip に手動で ComicInfo.xml を追加したので反映したい
- ページや動画を差し替えたので pageCount / hasVideo / サムネを更新したい
- サムネ生成が壊れた or 過去仕様で生成されたものを作り直したい
- 動画対応追加のように機能追加で既存書籍のメタデータを再検出したいが、全件走査するほどでもない

現在は「全件再インデックス」を実行するしかなく、800 冊規模で数十秒〜分待つ必要がある。

## 現状の挙動

- ビューワーのメニューシートに書籍単位の操作は「お気に入りトグル」「表紙設定」「既読にして閉じる/未読に戻す」しかない
- 個別書籍のキャッシュ破棄は API (`LibraryService#invalidateBookCache`) には存在するが、ユーザーから呼び出す UI がない

## 変更後の挙動

### ビューワー内の操作

- メニューシート下部 (「既読にして閉じる」ボタンの近く) に **「この本を再インデックス」** ボタンを追加
- タップすると即座に実行 (確認ダイアログなし)
- 実行中はボタンが disabled、ラベルを「再インデックス中…」に変更、スピナー系の視覚フィードバック
- 完了で「再インデックスしました」トースト (既存のトースト UI を流用)
- 完了時にビューワーの表示内容 (ページ・メタパネル・サムネのプリフェッチ) を新しいメタデータで再描画
- 失敗時は「再インデックスに失敗しました」トーストを出し、ボタンを戻す

### 再インデックスで行うこと

以下をこの順で全て実行 (「すべてのキャッシュを破棄」):

1. **ページキャッシュ破棄** — `PageCache.invalidate(bookId)` (メモリ manifest + 開いた ZipReader + ディスクの `<baseDir>/pages/<bookId>/`)
2. **サムネキャッシュ破棄** — `<baseDir>/thumbs/<bookId>.{webp,jpg,…}` を削除
3. **書籍メタデータ再取込** — ファイルシステムから size/mtime を再取得し、mode="full" 相当で books テーブルを upsert (差分判定を必ず skip する)
4. **ComicInfo.xml 再取込** — 存在すれば `comic_info` を上書き、なければ既存レコードを削除
5. **hasVideo 再検出** — zip 内の動画エントリ有無を再判定
6. **pageCount リセット** — `null` に戻し、次のページ一覧取得時に埋め直させる (既存の遅延埋め機構を再利用)

### 保持されるユーザー状態

以下は **book.id が同じであれば触らない**:

- 既読状態 (`read_states`)
- お気に入り (`favorites`)
- 表紙ページ設定 (`book_covers`) — ただし新ページ数の範囲外なら自動リセット (既存の `deleteCoverIfOutOfRange` を流用)

### ファイル自体が消失していた場合

再インデックス実行時にファイルが存在しなければ、404 を返してトーストで「ファイルが見つかりません」と表示。DB からは削除しない (通常の全件再インデックスと同じ挙動)。

### API

- **新規: `POST /api/books/:id/reindex`**
  - 成功: 200 と更新後の book / comicInfo を返す
  - ファイル欠損: 404
  - パス解決失敗 (root 消失等): 404
  - その他: 500

一覧側 (`/api/books/sections` や `/api/books`) は次回取得時に新しい値になる。既に開いている一覧タブへの自動反映はしない (BFCache 復帰時に取得し直す既存の仕組みで十分)。

## スコープ外

- 一覧画面のカードからの再インデックス操作 (長押し/右クリック等)
- 複数書籍の一括再インデックス (対象を選んで実行)
- 進捗表示 (通常は 1 秒以内で完了するため不要)
- ディレクトリ単位の再インデックス
- 確認ダイアログ (「不要」で確定)

## E2E テストチェックリスト

- ### 正常系
  - [ ] ビューワーのメニュー内に「この本を再インデックス」ボタンが表示される
  - [ ] タップで実行され、完了トーストが出る
  - [ ] ComicInfo.xml を差し替えた zip を再インデックスすると、メタパネルのタイトル/作者/タグが新しい内容に更新される
  - [ ] 画像しかなかった zip に mp4 を足して再インデックスすると、動画ブックとして扱われ、開き直しで動画が再生される
  - [ ] 通常本を再インデックスしても既読・お気に入り・表紙設定が保持される
- ### 異常系
  - [ ] 再インデックス中にファイルが消えていた場合、失敗トーストが出てビューワーはクラッシュしない
  - [ ] 存在しない book id に対する API 呼び出しは 404
- ### エッジケース
  - [ ] 表紙ページ設定が新ページ数の範囲外になったら自動的にリセットされる
  - [ ] 動画ブック → 通常本、通常本 → 動画ブックの相互変換が正しく反映される (hasVideo フラグ・ページ構成)
  - [ ] 実行中に連打してもリクエストは 1 回だけ発火する (ボタン disabled)
