# お気に入り機能

## 背景

書籍の中には「また読みたい」「特に気に入っている」ものがあり、 読了済み
かどうかとは別に印を付けたい。 現状の読書状態 (未着手 / 読書中 / 読了)
では表現できないため、 独立した「お気に入り」の概念を導入する。

## 現状の挙動

- 書籍に付けられる印は読書状態 (lastPage / finished) のみ
- 一覧画面では「続きから / 最近追加した / 最近読んだ」セクションがあるが、
  「お気に入り」はない
- カード上のバッジは「既読」 / 「読書中 (Np〜)」 のみ

## 変更後の挙動

### データモデル

書籍単位の boolean フラグとして実装。 読書状態とは独立。

DB スキーマ v4 で新テーブルを追加:

```sql
CREATE TABLE favorites (
  book_id    INTEGER PRIMARY KEY,
  created_at INTEGER NOT NULL,
  FOREIGN KEY (book_id) REFERENCES books(id) ON DELETE CASCADE
);
CREATE INDEX idx_favorites_created_at ON favorites(created_at);
```

- ON DELETE CASCADE で書籍削除と連動 (既存の read_states と同じ運用)
- `read_states` に列追加せず別テーブルにしておくことで、 「お気に入りだが
  未着手」 「読了済みだが お気に入り」 のような組み合わせを単純に扱える
- マイグレーション失敗対策として書籍テーブル変更時の foreign_keys OFF
  ルールを徹底 (前回の事故の再発防止)

### API

#### POST `/api/books/:id/favorite`

書籍をお気に入りトグル。 リクエストボディ:

```json
{ "favorited": true }
```

レスポンス:

```json
{ "favorited": true, "createdAt": 1782640000 }
```

- 404: book_id が存在しない
- 400: `favorited` が boolean でない
- 既に同じ状態を要求した場合も 200 (idempotent)
- `createdAt` は favorited=true 時のみ意味を持つ (false 時は null)

#### 既存 API のレスポンス拡張

- `GET /api/books`: `book.favorited: boolean` を追加 (LEFT JOIN favorites)
- `GET /api/books/:id`: 同上
- `GET /api/books/sections`: 当初追加予定だった `favorites: [...]` は採用
  しない (トップカルーセルへの掲載をやめたため)
- `GET /api/books?favorited=1`: お気に入りのみに絞り込み
- `GET /api/books?sort=favorited`: お気に入りを先頭、 その後タイトル順

### URL クエリ

- `?favorited=1` を追加 (既存の `?q=` / `?root=` / `?directory=` /
  `?status=` と組み合わせ可能)
- `?sort=favorited` (タイトル順との二段ソート)
- BFCache 復元時も `?favorited=` を再同期する (URL ↔ state 同期に組み込む)

### UI: サイドバー

「すべて」の真下に「★ お気に入り」リンクを常時表示 (件数バッジ付き)。

```
[ライブラリ]
  すべて
  ★ お気に入り  (N)
▼ Comics
  ...
▼ Pixiv
  ...
```

- タップで `?favorited=1` で絞り込み
- 件数は API `/api/config` または `/api/directories` のレスポンスに
  `favoritesCount` を追加して取得

### UI: トップカルーセル

トップカルーセルへの「お気に入り」セクション追加は当初検討したが、
カルーセルが縦に伸びすぎるため **取りやめ**。 お気に入り一覧はサイドバーの
「★ お気に入り」リンク経由で見る運用に絞る (`?favorited=1` で全件)。

参考: `repository.listRecentlyFavorited` 関数は将来再導入する余地のため
残してある。 現在 `/api/books/sections` レスポンスには含まれない。

### UI: カード上のお気に入り表示

カード右上に状態アイコンを表示:

- **お気に入り済み**: 黄色の `★` アイコンを常時表示
- **未設定**: 何も出さない

既存の「既読 / 読書中」バッジは左下のままで、 ★ は右上で住み分け。

### UI: トグル操作 (3 経路)

#### (1) ビューワーのメニューシート

メニュー内のメタパネル冒頭に「★ お気に入り」トグルボタンを追加。

- 押下で即座に POST し、 アイコンと文言 (★/☆ / 「お気に入り中」/「お気に入りに追加」) が反転
- 失敗時は元に戻して toast (既存パターンに合わせる)

#### (2) カード右上の ★ ボタン (常時表示)

タッチ環境では右上の ★ ボタンを常時表示し、 タップで toggle:

- お気に入り済み: 黄色 ★ がはっきり表示
- 未設定: 半透明な ☆ で、 場所だけ占有しタップ可能

長押しコンテキストメニューは **採用しない** (スクロール中の誤爆を避ける
ため)。 「タップ = ビューワー遷移」 「★ ボタンタップ = toggle」 の単純な
住み分けにする。 ★ ボタンのクリックイベントはカード遷移を `stopPropagation`
で止める。

#### (3) デスクトップ向けのホバー表示

`(hover: hover)` メディアクエリで分岐し、 マウス利用環境では:

- 未設定の ☆ は通常非表示、 カードホバー時 / フォーカス時にフェードイン
- お気に入り済みの ★ は常時表示
- 右クリック (contextmenu) でも toggle する

タッチ環境ではこのフェードルールは無効化され、 すべてのカードで ★ ボタンが
常時可視。

### UI: ソート / フィルタとの組み合わせ

- ツールバーのソート select に **「お気に入り優先」** 選択肢を追加
  (既存の タイトル / 更新 / 追加 / 未読優先 と並べる)
- ツールバーに **「★ お気に入りのみ」 チェックボックス** を追加。 status
  select とは独立、 同時適用可能
  - 「お気に入り × 読書中」 「お気に入り × 読了前のみ」 等の組み合わせ
  - URL は `?favorited=1&status=reading` のように両方乗る
- 「条件をクリア」 ボタンは `?favorited=` も含めてリセットする

### 一覧の空状態

- お気に入りフィルタ ON で 0 件 → 「お気に入りに追加した本はまだありません」
  と表示し、 「条件をクリア」 ボタンを出す (既存の空状態の文言切り替えに従う)

### サムネキャッシュ / 既読状態への影響

- なし。 favorites は独立テーブルで、 既存の read_states / page_cache /
  thumbnails に変更を加えない

## スコープ外

- お気に入りをシリーズ / 作者単位で付けること (将来検討)
- お気に入りリストのインポート / エクスポート
- お気に入りに対するメモやタグ
- 複数ユーザー / 共有用のお気に入り (認証機構自体がない)
- 長押しコンテキストメニューに「作者で絞り込み」「ビューワーで開く」等の
  追加項目 (今回はお気に入りトグルのみ。 メニュー枠組みは将来拡張可能に
  作る)
- お気に入りカード上のドラッグ並び替え (created_at DESC 固定)

## E2E テストチェックリスト

### 正常系

- [ ] ビューワーのメニューシートで `★ お気に入り` ボタンを押すと
      `/api/books/:id/favorite` に POST され、 アイコンが反転する
- [ ] スマホで一覧カードの右上 ★ ボタンが常時表示され、 タップで toggle
      でき、 カード本体タップは引き続きビューワーへ遷移する
- [ ] デスクトップ環境でカードにホバーすると右上に ☆/★ ボタンが出て、
      クリックでトグルできる (お気に入り済みは常時表示)
- [ ] デスクトップで右クリックしても toggle される
- [ ] サイドバーの「★ お気に入り」リンクをタップすると `?favorited=1` で
      URL が更新され、 お気に入り書籍のみが一覧に出る
- [ ] サイドバーのお気に入り件数バッジが現在の件数と一致する
- [ ] ソート「お気に入り優先」を選ぶと、 お気に入り済みが先頭、 その後
      タイトル順で並ぶ
- [ ] `?favorited=1&status=reading` 等、 status と組み合わせて絞り込める

### 異常系

- [ ] 存在しない book_id に POST `/favorite` すると 404
- [ ] 不正なボディ (favorited が boolean でない / 欠落) で 400
- [ ] DB 書き込み失敗時、 UI はエラー表示し元の状態に戻る (楽観更新の
      ロールバック)
- [ ] 同じ状態を再度トグルしようとしても idempotent (200)

### エッジケース

- [ ] お気に入り済みの書籍がインデックスから削除されると、 favorites
      レコードも CASCADE で消える (一覧に出ない、 サイドバー件数も更新)
- [ ] 別 root に同名相対パスの書籍があってもお気に入り状態は独立
      (favorites は book_id 単位)
- [ ] iOS Safari でカードを長押ししても画像保存メニューが出ない
      (`-webkit-touch-callout: none` が効いている)
- [ ] スクロール中に ★ ボタン上を通過しても toggle が誤発火しない
      (タップ判定 = pointerup までの移動が小さい時のみ発火)
- [ ] BFCache で戻ったときに `?favorited=1` が URL から復元される
      (iOS Safari)
- [ ] PWA standalone モードでもホバーボタンが出ない (タッチ環境扱い)
- [ ] DB マイグレーション v3 → v4 が走った後も既存の read_states /
      comic_info が消えない (foreign_keys OFF を migrate 中だけ徹底)
