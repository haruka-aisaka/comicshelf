# 再インデックス: 確認ダイアログ + 差分インデックス

## 背景

現在の「今すぐ再インデックス」 はボタン押下と同時に **全件再インデックスを実行**する。
ライブラリには 800+ 冊あり、 各 ZIP を開いて ComicInfo.xml を抽出するため
1 回の所要時間は数分〜十数分。 タップミスで実行されると数分間ライブラリの
レスポンスが落ち、 サーバの CPU/IO も高負荷になる。 加えて、 実際には大半の
書籍は前回から変わっておらず、 毎回全件処理するのは無駄が大きい。

そのため:
1. 重い処理であることをユーザーに伝え、 ワンクリックで暴走しないようにしたい
2. 普段使いは「変わったファイルだけ取り込む」 差分インデックスで済ませたい

## 現状の挙動

設定画面:
- 「今すぐ再インデックス」 ボタンを押すと、 確認なしで全件再インデックスが開始
- 800+ 冊で 8〜10 分かかる (前回の最終実行時間で確認済み)
- 各 ZIP を毎回開いて ComicInfo.xml を抽出 → 大半は変化なしで無駄

自動実行:
- 起動時 + `watchInterval` (デフォルト 3600 秒 = 1 時間) ごとに全件再インデックス
- 同様に毎回 800+ 冊を全部開く

## 変更後の挙動

### ボタンと確認ダイアログ

設定画面の「今すぐ再インデックス」 ボタンを押すと、 中央に確認ダイアログが
出る。 内容:

```
┌─ インデックス方法を選んでください ─────────────────────┐
│                                                        │
│  ライブラリ全 851 冊から、 変更があったファイルだけを   │
│  処理するか、 全件を処理するかを選べます。              │
│                                                        │
│  [ 差分インデックス (推奨) ]                            │
│  → 前回から変わっていない本はスキップ。 数秒で完了    │
│                                                        │
│  [ 全件再インデックス ]                                 │
│  → 全 ZIP を開き直す。 前回は 8 分 32 秒かかりました   │
│                                                        │
│  [ キャンセル ]                                         │
│                                                        │
└────────────────────────────────────────────────────────┘
```

- **差分** はデフォルトで推奨。 タップ→即時開始
- **全件** はリスクを示す情報 (前回所要時間) を併記してから開始
- **キャンセル** は何もせずダイアログを閉じる
- 既に実行中の時はボタン押下時に「既に実行中です」 と表示してダイアログは出さない

### 差分インデックスの判定基準

各書籍について、 ファイルスキャン時点の `(path, sizeBytes, modifiedAt)` を
DB の `books` テーブルの既存レコードと比較する:

- **3 つすべて一致** → 変更なしと判定し、 ZIP を開かずスキップ
  - `books` テーブルは触らない (`addedAt`/`modifiedAt` も保持)
  - `comic_info` も再取得しない (前回値を維持)
- **1 つでも不一致 / 新規 path** → 変更ありと判定し、 通常の処理を行う
  - `upsertBook` で books 更新
  - ZIP を開いて ComicInfo.xml を再パース → DB に反映

削除検出 (= スキャンで見つからなくなった path) は差分・全件問わず実施。

### 全件再インデックスの判定基準

差分判定をスキップし、 すべての書籍について ZIP を開いて再パース。
現状と同じ挙動。

### 自動インデックス (watchInterval)

- **差分モードで動く**
- 起動時 1 回 + watchInterval ごと、 すべて差分判定を経由
- ユーザーが手動で「全件」 を選ばない限り、 全 ZIP を開く処理は走らない

### 進捗バナーの表示

差分インデックスでは大半が skip される。 バナーの詳細欄を更新:

```
インデックス中                        12 秒
[==========================>            ]
145 件処理済み / 約 851 件 ・ 変更 3 件 ・ ComicInfo 1 件
処理中: by-author/foo/bar.zip
```

- **N 件処理済み**: scan を通過した総数 (skip 含む)
- **変更 M 件**: 実際に DB を更新した数 (upsertBook が走った数)
- ComicInfo は変更検知された書籍に対してのみ再パースされる

完了バナーも同様に「変更 M 件 / ComicInfo K 件」 を出す。

### API 形式の変更

`POST /api/index/rebuild`:
- 新規クエリパラメータ `?mode=incremental` (デフォルト) / `?mode=full`
- 旧来の呼び出し (パラメータなし) は **互換性のため `incremental` 扱い**
  - これまで「全件」 だった挙動から **デフォルトが差分に変わる** が、 通常ユーザーには望ましい変化
  - 全件を必要とするテスト / スクリプトは明示的に `?mode=full` を付ける

`IndexStats` / `CurrentRunStatus` / `IndexerRunResult` に `skipped: number` を追加 (= 差分で skip された件数)。

## スコープ外

- 差分判定アルゴリズムをハッシュベース (sha256 等) に高度化 — 今回は mtime/size のみ
- 削除のみを検出する超軽量モード — 差分インデックスで十分高速なので不要
- ComicInfo.xml だけ変わって ZIP の mtime が変わらないケースの検知 — レアケース、 必要なら全件で対応
- 確認ダイアログのキーボードショートカット (Esc/Enter) — Nice to have
- 進捗バーの「変更検知率」 などの統計可視化

## E2E テストチェックリスト

### 正常系

- [ ] 「今すぐ再インデックス」 ボタン押下で確認ダイアログが出る
- [ ] ダイアログに「差分インデックス (推奨)」「全件再インデックス」「キャンセル」 の 3 ボタンがある
- [ ] 「全件再インデックス」 の説明文に **前回の所要時間** が表示される
- [ ] 「差分インデックス」 を選ぶと `POST /api/index/rebuild?mode=incremental` が呼ばれる
- [ ] 「全件再インデックス」 を選ぶと `POST /api/index/rebuild?mode=full` が呼ばれる
- [ ] 差分インデックス完了後、 何も変更がないライブラリでは `skipped == 全件数`、 `upserted == 0`
- [ ] 1 ファイルだけ mtime を変更して差分実行すると、 `skipped == 全件数-1`、 `upserted == 1`
- [ ] 全件再インデックスでは `skipped == 0`、 `upserted == 全件数`
- [ ] 進捗バナーの詳細欄に「N 件処理済み / 約 M 件 ・ 変更 K 件」 が出る
- [ ] 自動 (watchInterval) reindex も差分で動く (起動時に skipped > 0 になる)

### 異常系

- [ ] reindex 実行中にボタンを押すと「既に実行中です」 と表示され、 ダイアログは開かない
- [ ] `mode` パラメータに不正な値 (`?mode=xyz`) を渡すとデフォルト (`incremental`) として扱われる
- [ ] サーバが落ちている時にボタン押下 → エラーメッセージ表示
- [ ] ダイアログを開いた直後に自動 reindex が始まった場合、 ダイアログ内で「実行中」 状態が反映されるか、 ボタンが disabled に

### エッジケース

- [ ] 初回起動 (lastResult が null) で全件を選んだとき、 「前回の所要時間」 は「— (初回)」 などで代替表示
- [ ] 削除のみ (ファイルが消えた) でも差分インデックスが `removed` を検出して反映する
- [ ] サイズ 0 / mtime 0 のファイルでも差分判定が正しく動く
- [ ] ファイル名 (path) が変わっただけ (リネーム = 旧パス消失 + 新パス出現) の場合、 旧パスは removed、 新パスは upserted (= 新規追加扱い、 ComicInfo 再取得)
- [ ] mtime は同じだが size が変わったファイル → 「変更あり」 と判定される
- [ ] mtime / size 同じだが ComicInfo.xml だけ変わった場合 → skip される (これは仕様、 全件再インデックスで対応)
- [ ] 確認ダイアログを開いたままページを離れる/リロードしても破綻しない
- [ ] ダイアログの背景 (backdrop) クリックでキャンセル扱いになる
- [ ] スマートフォン幅 (402px) でダイアログが画面内に収まり、 各ボタンが押せる
