# トップバーをシンプル化: 絞り込みをポップオーバーに集約

## 背景

トップバーが、 ハンバーガー / タイトル / 検索バー / status フィルタ select /
sort select / ★お気に入りのみチェック / 設定アイコン と要素が多く、 とくに
モバイルで横幅に収まらず 2〜3 段に wrap してしまう。 結果:

- 一覧の表示領域が押し下げられる
- タイトル「comicshelf」 はモバイルで見えない
- 視線の集中先 (= 検索バー) が他のコントロールに埋もれる

「絞り込み」 は使う頻度こそあるが、 常時 select として並べる必要はない。
1 段のすっきりしたトップバーにまとめ、 絞り込みは明示的に開く UI にしたい。

## 現状の挙動

- トップバー: `☰ | comicshelf | 検索バー | status select | sort select | ★ | ⚙`
- モバイル (< 768px): 検索バー以降が wrap してトップバー全体が 2-3 段
- 「お気に入りのみ表示」 はトップバーのチェックボックス
- フィルタ系は値変更で即時適用される (debounce なし)

## 変更後の挙動

### レイアウト

トップバーは **モバイル / デスクトップ共通**で 1 段に統一:

```
┌────────────────────────────────────────────────┐
│ ☰  comicshelf   [🔍 検索...]   [▼ 絞り込み]  ⚙ │
└────────────────────────────────────────────────┘
```

- `☰` ハンバーガー (モバイルのみ表示、 既存)
- `comicshelf` タイトル (デスクトップで表示、 モバイルでは非表示 = 既存挙動)
- 検索バー (`flex: 1` で残り幅を埋める。 主役)
- **「絞り込み」 ボタン** (新規追加。 後述)
- `⚙` 設定アイコンへのリンク (既存)

(status / sort / ★ お気に入り のチェックボックスはトップバーから消える)

### 「絞り込み」 ボタン

- 配置: 検索バー右隣、 設定アイコン左
- アイコン: フィルタ (`▼` か漏斗 SVG)。 ラベル「絞り込み」 をデスクトップでは
  併記、 モバイルではアイコンのみ (ツールチップ "絞り込み")
- 押下でポップオーバーを開閉 (アンカーは絞り込みボタン自身)

#### アクティブ表示 (= 何らかの絞り込み中なら強調)

以下のいずれかが default 以外なら **ボタンを accent 色に**:

- status !== "all"
- sort !== "title"
- favorited === true

(現在の有効フィルタ数の数字バッジは出さない。 シンプルさ優先)

### ポップオーバー

- 位置: ボタン直下、 right-aligned (画面右にはみ出さない範囲で)
- 幅: 240-280px 程度。 モバイルでは画面右端から少しマージン
- 内容 (上から):
  1. **表示ステータス** (status select、 既存と同じ選択肢)
  2. **並び順** (sort select、 既存と同じ選択肢)
  3. **★ お気に入りのみ表示** (checkbox + ラベル)
  4. 区切り線
  5. **すべてリセット** ボタン (default に戻す。 トップバーから消える
     「条件をクリア」 の代替動線)
- 閉じる手段: 外側タップ (backdrop) / Esc / 絞り込みボタン再押下

#### 適用タイミング

- select / checkbox 変更で **即時適用** (現状挙動と同じ)。 ポップオーバーは
  open のまま維持して、 複数値の連続変更がしやすい
- 「すべてリセット」 押下時は default に戻し、 ポップオーバーは閉じる
- popstate (= スワイプバック) で URL の絞り込み状態が変わったら、 ポップ
  オーバー内のコントロール表示も同期する

### 履歴への積み方

- status / sort / ★ の変更は **従来通り replaceState** (連続変更で履歴が
  膨らまないように)。 これは現状挙動を維持
- 「すべてリセット」 は明示的な navigation アクションなので **pushState** で
  履歴に積む (戻る操作で元のフィルタに戻れる)

### モバイルでの追加考慮

- ポップオーバーは画面右上端 (= ボタン直下) から開く
- iOS Safari の native select オーバーレイは下部から開くが、 これは OS の
  挙動なのでそのまま (ポップオーバーの上にネイティブ select picker が重なる
  形)

### サイドバーへの絞り込み移設は不採用

候補に挙がっていた「サイドバーに絞り込みセクションを足す」 案は採用しない。
理由:
- サイドバーはディレクトリ・ルート・★お気に入りタブ (= **集合の選択**) の
  ナビゲーションが主役
- status / sort は「現在見ている集合に対する絞り込み」 で文脈が違う
- トップバーの絞り込みボタンの方が「コンテンツに対する操作」 として直感的

## スコープ外

- 件数バッジ (絞り込み数表示) の追加
- 絞り込みプリセット保存 (例: 「未読のみ + タイトル順」 を 1 タップで)
- 検索 input 内へのチップ形式統合 (`status:unread` などタグエディタ風)
- ポップオーバーのアニメーション凝った演出 (fade in/out のみ)
- デスクトップで select をそのまま並べる別 UI への分岐 (= 完全統一)

## E2E テストチェックリスト

### 正常系

- [ ] モバイル (390px) でトップバーが 1 段に収まる
- [ ] デスクトップ (1280px) でもトップバーが 1 段に収まる
- [ ] 検索バーが残りの幅を埋めて主役として表示される
- [ ] 「絞り込み」 ボタンを押すとポップオーバーが開き、 status / sort /
      お気に入り / リセットが見える
- [ ] status を変更すると即座に一覧に反映される (ポップオーバーは open のまま)
- [ ] sort を変更しても同じく即時反映
- [ ] お気に入りチェックを on にすると即時絞り込み
- [ ] 外側 (backdrop) をタップするとポップオーバーが閉じる
- [ ] Esc キーでもポップオーバーが閉じる
- [ ] 「絞り込み」 ボタンを再度押すと閉じる
- [ ] 「すべてリセット」 を押すと default 状態に戻り、 一覧が再ロードされ、
      ポップオーバーが閉じる
- [ ] いずれかが default 以外の時、 絞り込みボタンが accent 色になる
- [ ] 全部 default に戻すと、 ボタンの色も元に戻る

### 異常系

- [ ] popstate (戻る操作) で URL が変わった時、 ポップオーバー内 select も
      同期して表示される
- [ ] ポップオーバー open 中にウィンドウサイズを変えても破綻しない

### エッジケース

- [ ] iOS Safari で native select の picker (下から出るやつ) を開いた状態で
      ポップオーバー外側をタップしても、 ポップオーバーは閉じない (= select
      picker のタップが backdrop に伝わらない)
- [ ] PWA standalone でも同様に動作
- [ ] BFCache 復元時、 ポップオーバーは閉じた状態で復元される
- [ ] サイドバー (☰) が開いている状態で絞り込みボタンを押した時、 両方
      開かない (どちらか片方で十分)
