# 読了モーダル整理 + 自動送り「有効状態」 の永続化

## 背景

- 「閉じる時に既読にする」 のはユーザーから見て当然の挙動なので、 モーダル
  の主 CTA が「既読にして閉じる」 だと冗長。 短く「閉じる」 だけにしたい。
- 同じ流れで「お気に入りに追加して閉じる」 という頻出操作を 1 タップで
  完結できるボタンが欲しい。 現状はメニューシートを開いて ★ をタップして
  から閉じる、 と 2 段階必要。
- 自動ページ送り (auto-advance) は設定値 (秒数) は localStorage 永続化
  されているが、 「動作中だったか / 停止だったか」 は永続化されておらず、
  本を開くたびに「停止」 から始まる。 連続して別の本を開く時に毎回手動で
  「再開」 を押すのが手間。

## 現状の挙動

### 読了モーダル

最終ページで「次へ」 操作すると以下のモーダルが出る:

```
最後のページです
この本を既読にして閉じますか?
[キャンセル] [既読にして閉じる]
```

- 主 CTA「既読にして閉じる」 → `finished=true` 保存 → トップ遷移
- 「キャンセル」 → モーダル閉じてビューワー継続

### 自動送り

- `localStorage.comicshelf.autoAdvanceSec` に秒数だけ永続化
- 「動作中 / 停止」 (= `userStopped`) は永続化されず、 起動時は **常に
  停止**から始まる (`auto_advance.js` の `userStopped: true` 初期値)
- メニューシートで再生ボタン (▶ 再開) を押すと動き出す

## 変更後の挙動

### 読了モーダルの文言とボタン構成

```
最後のページです
この本を閉じますか?

[ 閉じる ]                    ← 主 CTA (緑系プライマリ)
[ お気に入りに加えて閉じる ]   ← 未お気に入りの時のみ表示
[ キャンセル ]                ← グレー
```

- **縦並び**の 3 ボタン (モバイルでタップしやすさ優先)
- 「お気に入りに加えて閉じる」 は **未お気に入りの本のときだけ表示**。 既に
  お気に入り済みなら隠す (シンプル化、 重複動線を排除)
- モーダル本文も「既読にして閉じますか?」 → 「閉じますか?」 に変更
  (「閉じる = 既読」 は当然なので明示しない)

#### 各ボタンの挙動

| ボタン | 挙動 |
|---|---|
| 閉じる | `POST /progress {lastPage: 最終, finished: true}` → トップへ |
| お気に入りに加えて閉じる | `POST /favorite {favorited: true}` + `POST /progress` を並列実行 → トップへ |
| キャンセル | モーダルを閉じてビューワー継続 |
| backdrop タップ / Esc | キャンセル相当 |

### 既読本でモーダルを出した場合

(現在の挙動を維持。 既読本でも閉じる動線としてモーダルを再利用する)

- 「閉じる」 (= 既読再送信 + トップ): idempotent
- 「お気に入りに加えて閉じる」 (未お気に入りの時のみ): お気に入り化 + トップ
- 「キャンセル」: ビューワーに戻る

### 自動送り「動作中」状態の永続化

- 新規 localStorage キー `comicshelf.autoAdvanceActive` (boolean) を導入
- ユーザーが **再生 / 停止 をトグルした時に書き込む** (= 「再開」 押下で
  true、 「停止」 押下で false)
- ビューワー起動時にこのキーを読み、 true なら自動送りを開始状態にする
  (= `userStopped: false` 相当)。 false / 未設定なら従来通り停止から
- スコープは **全本共通**: book_id ごとに分けない
  (使い分ける需要は薄く、 設定が散逸する方が混乱する)
- メニュー表示中の `systemPaused` (= 一時停止) は永続化対象外。 メニューを
  閉じれば自動的に再開する従来挙動を維持

#### 起動シーケンス

1. ビューワーロード時 (`AutoAdvance.updateUi()` 前)
2. `comicshelf.autoAdvanceActive` を読む
3. `true` なら `state.userStopped = false` で初期化し、 タイマー開始
4. `false` or 未設定なら従来通り停止状態

### 視覚的フィードバック

- 自動送り開始時のメニュー UI: ▶ / 一時停止アイコンが既存の連動ロジックで
  自動的に正しい状態を表示する (追加実装不要)

## スコープ外

- 自動送りの秒数を本ごとに変える機能 (今回は秒数も全本共通のまま)
- お気に入りボタンに加えて「シリーズを既読化」 等の追加操作
- 読了モーダルにメモ / 評価入力欄
- 自動送りのプリセット (5/10/15 秒ボタン等)
- 既読本での「お気に入りに加えて閉じる」 押下時に「未読に戻す」 を考慮する
  特別処理 (= 既読のままで favorite だけ追加)

## E2E テストチェックリスト

### 正常系

- [ ] 未読 + 未お気に入りの本で最終ページ → モーダルに 3 ボタンが縦に並ぶ
      (閉じる / お気に入りに加えて閉じる / キャンセル)
- [ ] モーダル本文が「この本を閉じますか?」 になっている
- [ ] 「閉じる」 を押すと `finished=true` で保存され、 トップへ遷移
- [ ] 「お気に入りに加えて閉じる」 を押すと favorite=true + finished=true が
      送られ、 トップへ遷移
- [ ] 既にお気に入り済みの本では「お気に入りに加えて閉じる」 ボタンは表示
      されない (2 ボタン: 閉じる / キャンセル)
- [ ] 既読本を開いて最終ページ → モーダルが出て、 「閉じる」 でトップへ
- [ ] 自動送りを「再開」 にした状態でビューワーを閉じ、 同じ本を再度開く
      → 自動送りが自動的に開始される
- [ ] 別の本を開いた場合も自動送りが自動的に開始される (全本共通スコープ)
- [ ] 自動送りを「停止」 にしてから閉じる → 次回は停止状態で開く
- [ ] 自動送りの秒数設定は従来通り永続化される (既存挙動の維持)

### 異常系

- [ ] 「お気に入りに加えて閉じる」 で favorite API がエラーでも progress
      は保存される (両方並列実行のため、 片方失敗しても他方が成立)
- [ ] favorite API 成功 + progress API 失敗の場合は toast でエラー、
      画面は留まる (= 楽観更新ロールバックして再操作を促す)
- [ ] `comicshelf.autoAdvanceActive` の値が不正 (`"foo"` 等) でも
      停止状態 (= false 扱い) で立ち上がる

### エッジケース

- [ ] iOS Safari で localStorage が無効 (プライベートブラウズ等) でも
      クラッシュせず、 デフォルト動作 (= 停止から開始) になる
- [ ] BFCache で戻ったときも自動送り状態は localStorage を再読み込みする
- [ ] メニューを開くと自動送りが一時停止 (`systemPaused`)、 閉じると再開
      する従来挙動が崩れていない
