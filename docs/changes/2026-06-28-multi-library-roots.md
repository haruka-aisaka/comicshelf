# 複数ライブラリルート対応

## 背景

`config.json` の `library.roots` は配列を受け付ける形になっているものの、
実態は単一ルート前提で実装されており、 README の「既知の制約」にも
「複数の library root を設定した場合、 ルート間で相対パスが衝突すると
レコードが上書きされる」と明記されている。

具体的な不具合:

- `books.path` は相対パスのみで UNIQUE のため、 異なるルートに同じ
  相対パスの ZIP があると後勝ちで上書きされる
- ディレクトリツリーはルートをまたいで同名ディレクトリを単一ノードに
  統合してしまう
- ライブラリの正体 (どの root に属する書籍か) が UI から識別できない

`/mnt/nas/Comics` (既存) に加えて `/mnt/nas/Public/PixivLibrary` を追加する
ニーズが生じたため、 複数ルートを実用的にサポートする。

## 現状の挙動

- 設定: `library.roots` は配列だが、 複数指定すると衝突時に DB レコードが
  上書きされる
- インデックス: 全 root を順に走査するが、 DB 登録時にどの root 由来かを
  記録していない
- 一覧画面のディレクトリツリー: root をまたいで同名ディレクトリを 1 つの
  ノードに統合
- 検索 `?q=` / `?directory=` / ソート: 単一の相対パス空間しか想定して
  いない
- ビューワーのページ配信: `library.ts` が全 root を順に試して最初に
  見つかったファイルを返す (DB 側で上書きされているため矛盾が起こりうる)

## 変更後の挙動

### 設定ファイル

`config/config.json` の `library.roots` を **オブジェクト配列** にも
対応させる。 文字列のみの旧形式も互換のため受け付ける。

```jsonc
{
  "library": {
    "roots": [
      { "id": "comics", "name": "Comics", "path": "/mnt/nas/Comics" },
      { "id": "pixiv",  "name": "Pixiv",  "path": "/mnt/nas/Public/PixivLibrary" }
    ],
    "extensions": [".cbz", ".zip"]
  }
}
```

- `id`: 内部識別子 (英数 / `_` / `-`)。 DB の `root_id` に保存。 一度
  決めたら不変前提。 変更すると既読状態の紐付けが壊れる
- `name`: UI 表示名 (人間向けラベル)
- `path`: 実ファイルシステム上の絶対パス

旧形式 (`["/comics"]` のような文字列配列) を読み込んだ場合は、
`id` をパスから自動生成 (basename + サニタイズ)、 `name` も同じ値を
使い、 互換動作する。

### DB マイグレーション

起動時の自動マイグレーションで `books` テーブルに `root_id` カラムを
追加し、 UNIQUE 制約を `(root_id, path)` に変更する。

既存レコード (root_id が NULL) は **config.json で先頭に定義された
root** に自動的に紐付ける。 既読状態 (`read_state`) ・ ComicInfo ・
ページキャッシュは保持される。

config.json に書かれていない id を持つレコードがマイグレーション後に
出てしまった場合 (= ユーザーが root を削除した場合) は、 DB には残るが
一覧には現れない (orphan 扱い)。 起動時ログに警告を出す。

### 一覧画面のディレクトリツリー

サイドバーのディレクトリツリーは **ルートごとにセクションを分ける**。

```
[ライブラリ]
▼ Comics
  ▼ シリーズA
      第1巻
      第2巻
  ▼ シリーズB
▼ Pixiv
  ▼ user_12345
  ▼ user_67890
```

- 各ルートは `name` をラベルにした最上位の `<details>` として表示
- ルート行をタップすると、 そのルート配下を全件絞り込み
  (URL: `?root=<id>`)
- 配下の階層は従来どおり `<details>` で展開、 prefix 一致でフィルタ
- ルートが 1 つしかない場合は、 現状互換のためルート行を省略してフラットに
  表示する (UX 上の差分を最小化)

### URL / API パラメータ

- 一覧画面の URL に `?root=<id>` を追加 (任意)。 既存の `?q=` `?status=`
  `?sort=` `?directory=` と組み合わせ可能
- `?directory=` は **ルート内の相対パス** として解釈される。
  `?root=<id>` と組み合わせて初めて一意になる
- `?root=<id>` のみ指定: そのルート配下を全件
- `?root=` 未指定 + `?directory=` 指定: 後方互換のため、 該当する
  ディレクトリを全 root から探す (複数 root にマッチすれば全部表示)
- BFCache 復元 (戻る) で `?root=` も再同期する

API:

- `GET /api/books` に `?root=<id>` を追加
- `GET /api/directories` のレスポンスを以下に拡張:
  ```jsonc
  [
    { "rootId": "comics", "directory": "シリーズA", "count": 5 },
    { "rootId": "pixiv",  "directory": "user_12345", "count": 12 }
    // ...
  ]
  ```
  旧フォーマット (`{directory, count}`) の利用箇所はフロントのみのため、
  併走期間は設けず一括移行する
- `GET /api/config` に `roots: [{id, name}]` を追加 (`path` は返さない)
- `GET /api/books/:id` のレスポンスに `rootId` を含める

### ビューワー / 既読

- ビューワー内のメタパネルから「ライブラリ名」を **表示しない**
  (UX を増やしすぎない方針。 必要なら別 PR で検討)
- 既読状態 / 進捗 / キャッシュは `book.id` ベースなので影響なし。
  ただし `book.id` の生成ロジックが `root_id` を含むようになる
  ことに注意 (= 既存 ID は維持、 新規書籍の ID 生成にだけ影響)

### 設定画面

- `/settings.html` に **「ライブラリ」セクション** を追加し、
  config.json 上に設定された各 root の `name` / `path` / 当該 root に
  属する書籍数を読み取り専用で一覧表示する
- root の追加 / 削除 / 編集 はこの画面からはできない (config.json を
  直接編集する運用)

### 検索 / ソート / 「続きから」 / 「最近追加」

- 検索 `?q=` は全 root 横断のまま (変更なし)。 検索結果カードには
  ルートを示すバッジ等は **付けない** (UX 上の混乱を避けるため。
  パスを見れば分かる前提)
- ソート (タイトル / 更新 / 追加 / 未読優先) はすべて全 root 横断
- 「続きから / 最近追加 / 最近読んだ」セクションも全 root 横断

### インデクサー

- 再インデックス (差分 / 全件) は全 root を順次走査
  (現状と同じ)
- 進捗バナーの `currentFile` 表示は `<rootName>/<relativePath>` の形に
  する
- インデクサーは config.json に存在する root だけを走査する。 DB に
  残っている孤立 root_id のレコードはマイグレーション時に skip しつつ
  保持する

## スコープ外

- root の動的追加 / 削除を UI から行う機能 (将来検討)
- root をまたいだ書籍の移動 (検出 / 履歴移行) サポート
- 検索結果カードにライブラリ名バッジを表示すること
- root ごとの個別設定 (拡張子 / 既定読書方向の override 等)
- 認証や root 単位のアクセス制御
- マイグレーション失敗時の自動ロールバック (失敗したら DB をバックアップから
  戻す運用)

## E2E テストチェックリスト

### 正常系

- [ ] config.json に 2 つの root (Comics / Pixiv) を設定して起動すると、
      両方の root の書籍が一覧に表示される
- [ ] サイドバーのディレクトリツリーに root セクションが 2 つ表示され、
      それぞれの配下に階層が展開される
- [ ] ルート行をタップすると `?root=<id>` で URL が更新され、 そのルート
      配下の書籍のみが表示される
- [ ] root セクション配下の子ディレクトリをタップすると `?root=<id>`
      + `?directory=<path>` の両方が URL に乗り、 正しく絞り込まれる
- [ ] `?q=` 検索は両 root 横断で結果を返す
- [ ] `/settings.html` にライブラリ一覧 (name / path / 件数) が表示される
- [ ] `GET /api/config` のレスポンスに `roots: [{id, name}]` が含まれる
      (`path` は含まれない)
- [ ] ビューワーで両 root の書籍がそれぞれ正常に開ける (ページ配信が
      正しい root のファイルから行われる)

### 異常系

- [ ] 同じ相対パスを持つファイルが両 root に存在しても、 両方が別レコード
      として DB に保存され、 一覧に両方表示される
- [ ] 旧形式 (`"roots": ["/comics"]`) の config.json でも起動でき、
      自動生成された id で動作する
- [ ] config.json から root を 1 つ削除して起動すると、 該当 root の
      書籍は一覧に出ない (DB レコードは保持されるが orphan として skip)、
      起動ログに警告が出る
- [ ] config.json に同じ id が重複定義された場合、 起動時に明示エラーで
      落とす (= 黙って後勝ちにしない)
- [ ] id にサニタイズ不能な文字が含まれる場合、 起動時にエラー

### エッジケース

- [ ] 既存 DB (`root_id` カラムなし) を持つ状態で起動すると、 自動
      マイグレーションが走り、 既存レコードがすべて先頭 root の id に
      紐付けられる。 既読状態 / キャッシュは消えない
- [ ] root が 1 つしかない場合、 サイドバーのディレクトリツリーは
      従来どおりフラットに表示 (ルートラベル行は省略)
- [ ] BFCache で戻ったとき、 `?root=<id>` を含む URL から状態が
      正しく復元される (iOS Safari)
- [ ] `?directory=<path>` のみ (`?root=` なし) で複数 root に同名
      ディレクトリがあれば、 両 root の該当書籍がまとめて表示される
- [ ] 差分インデックス中に「処理中: Pixiv/user_12345/...」のように
      root 名付きで currentFile が表示される
- [ ] root 追加直後の全件再インデックスで、 既存 root のレコードは
      上書きされない (id ベースで一意になっているため)
- [ ] 再起動を挟んでもマイグレーションは一度だけ走り、 2 回目以降は
      no-op
