# zip 内動画ファイル (mp4/webm) の再生対応 (非公表機能)

## 背景

pixiv うごイラ等を変換したアーカイブ (例: `cover.jpg` + `ugoira.mp4` + `ComicInfo.xml` の zip) をライブラリに置いても、現状は動画が完全に無視され「cover.jpg 1 ページだけの本」になってしまう。動画をビューワーで再生できるようにしたい。

本機能は **README 等に記載しない非公表機能** とする。設定フラグは設けず常に有効。

## 現状の挙動

- インデクサは zip 内の画像拡張子 (jpg/jpeg/png/webp/gif/avif/bmp) のみをページとして認識し、`.mp4` 等の動画エントリは列挙段階で除外される。
- 例の zip は「1 ページ (cover.jpg) の本」としてインデックスされ、ビューワーでは静止画 1 枚が表示されるだけ。
- ページ配信 API は Range リクエスト非対応 (常に全バイト返却)。

## 変更後の挙動

### インデックス

- zip 内の `.mp4` / `.webm` エントリをページ候補として認識する。
- **動画を 1 つ以上含む本は「動画ブック」として扱い、ページは動画エントリのみで構成する** (ファイル名順)。zip 内の画像はページに含めない。
- 動画を含まない本は従来どおり (挙動変更なし)。
- 既存インデックス済みの本は、再インデックス (差分/全件) 後に動画ブックとして反映される。

### サムネイル

- 動画ブックのサムネイルは **zip 内の画像 (ファイル名順で先頭、例では cover.jpg) から生成** する。
- zip 内に画像が 1 枚もない動画ブックは、プレースホルダー (既存のサムネ生成失敗時と同じ見え方) とする。動画フレームからのサムネ生成は行わない。

### 一覧画面

- 動画ブックも通常の本と同様にカード表示される (ページ数 = 動画本数)。
- **動画ブックのサムネイル中央に再生バッジ (▶ の半透明丸アイコン) をオーバーレイ** し、動画であることが一目でわかるようにする。バッジは装飾のみで、タップは通常どおりカード (ビューワーへ遷移) に透過する。
- 動画ブック判定はインデックス時に記録する。本機能追加前にインデックス済みの動画ブックは、再インデックスするか一度ビューワーで開くとバッジが反映される。

### ビューワー

- 動画ページは `<img>` の代わりに動画プレイヤーで表示し、**表示と同時に自動再生・ループ・ミュート** で再生する (うごイラ的な体験)。ネイティブコントロールは表示しない。
- ページ送り (タップ左右端 / スワイプ / ← → / Space)、中央タップでのメニュー開閉、進捗バー、既読管理は画像ページと同じ操作体系。
- ページを離れたら再生を停止する (バックグラウンドで再生し続けない)。
- **動画ページは常に単ページ表示** — 見開きモードの設定に関わらず、見開きペアに入らない。
- **自動ページ送りは動画ページでは進めない** (カウンターを停止。動画ページを抜けて画像ページに戻れば再開)。
- ピンチ拡大・ダブルタップズームは動画ページでは無効 (通常サイズでの fit 表示のみ)。
- 「このページを表紙にする」操作は動画ページでは無効 (メニュー項目を非表示または disabled)。
- ローディング: 動画の初期バッファ中は既存のスピナーと同じ見え方。

### 配信

- ページ配信 API は動画エントリに対して正しい Content-Type (`video/mp4` / `video/webm`) を返す。
- 動画配信は **HTTP Range リクエストに対応** する (iOS Safari では Range 非対応だと再生できないため。ユーザー視点では「iPhone でも再生できる」が仕様)。

## スコープ外

- 音声再生 (常にミュート。音声トグル UI も追加しない)
- mp4/webm 以外の動画形式 (mkv, avi 等) と、コーデックのトランスコード (ブラウザが再生できない動画はそのまま再生失敗)
- 動画フレームからのサムネイル生成
- 画像ページと動画ページの混在構成 (動画があれば動画のみ)
- 設定画面・README への記載、feature flag
- シークバー等の再生コントロール UI

## E2E テストチェックリスト

- ### 正常系
  - [ ] 例の zip (cover.jpg + ugoira.mp4) をインデックスすると 1 ページの動画ブックになり、サムネは cover.jpg 由来
  - [ ] 一覧画面で動画ブックのサムネ中央に再生バッジが表示され、通常の本には表示されない
  - [ ] ビューワーで開くと動画が自動再生・ループ・ミュートで表示される
  - [ ] 動画 2 本以上の zip でページ送り (スワイプ/キー/タップ) が動作し、離れたページの動画は停止する
  - [ ] 中央タップでメニュー開閉、既読が最終ページ到達で付く (既存挙動と同じ)
  - [ ] 動画を含まない既存の本の一覧・閲覧に変化がない
- ### 異常系
  - [ ] ブラウザが再生できない動画 (壊れたファイル等) でもビューワーがクラッシュせず、ページ送りで抜けられる
  - [ ] 画像なしの動画のみ zip でサムネがプレースホルダーになる
- ### エッジケース
  - [ ] 見開きモード「常に見開き」でも動画ページは単ページ表示
  - [ ] 自動ページ送り ON で動画ページに到達するとカウンターが止まり、手動で画像ページに移ると再開する
  - [ ] Range リクエスト (`Range: bytes=0-`) に 206 で応答する
  - [ ] リロード/BFCache 復元後も動画ページが再度自動再生される
