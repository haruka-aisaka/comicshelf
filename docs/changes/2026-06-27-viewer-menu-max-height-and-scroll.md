# ビューワーのメニューシートに高さ上限 + メタパネルの内部スクロール

## 背景

ビューワー中央タップで開くメニューシート (`#menu-overlay`) は、
書籍が多くのタグを持つ場合にメタパネル (作者・シリーズ・タグ chip 群) が
縦に伸び、 **シートが画面のほとんどを覆ってしまう**。 「進捗を見ながら少しだけ
設定を確認する」 体験が損なわれる。

## 現状の挙動

- `.menu-overlay` は `position: fixed; bottom: 0;` で高さ自動 (中身に応じて伸縮)
- タグが 30+ ある書籍だとシートが viewport の 80-90% を埋める
- 背景のページがほぼ見えない
- 一覧画面側の sticky topbar も含めて視認性が下がる

## 変更後の挙動

### シートの高さ上限

- `.menu-overlay` に **最大高 = 画面の 70%** を設定
- これより内容が多い場合、 シート自身は伸びず、 内部スクロールで対応
- 上端 30% はビューワーの背景ページが見える状態を維持
- SafeArea (notch/bottom indicator) も考慮済み

### メタパネルの内部スクロール

- メタパネル (`#meta-panel`) を「シート内でスクロールする領域」 にする
  - 作者・シリーズ・タグ chip すべてを含む
- シート上部 (戻る/タイトル/ページ番号/×) と下部 (シーク・設定・既読化)
  はスクロールせず常に見える
- スクロール可能な内容がある場合、 メタパネルの **下端に fade gradient**
  を表示して「もっと下にある」 ことを示唆
- スクロールが最下端に達したら gradient は消える

### その他

- メニューを閉じて再度開いた時、 スクロール位置は **上端にリセット**
  (初見の状態に戻す方が自然)
- backdrop / Esc / 中央タップでの閉じる動作は変更なし

## スコープ外

- タグの並び順や色分け
- タグの個数制限 / 「+N more」 表示
- 作者/シリーズ部分の折りたたみ
- メニューシートの開閉アニメーション

## E2E テストチェックリスト

### 正常系

- [ ] タグが多い書籍 (30+) の viewer でメニューを開くと、 シートが viewport の
      **約 70% で頭打ち** になる
- [ ] シート上端から viewport の 30% 程度の高さでビューワーのページが見える
- [ ] メタパネル内でタグ chip を上下スクロールできる
- [ ] スクロールしてもシート上部 (戻る/タイトル/ページ番号/×) は固定で見え続ける
- [ ] スクロールしてもシート下部 (シーク・既読化ボタン) は固定で見え続ける
- [ ] スクロール可能な状態のときメタパネル下端に fade gradient が出る
- [ ] スクロールが最下端に達すると gradient が消える

### 異常系

- [ ] タグが少ない書籍 (5 個以下) ではメタパネルが伸び切らず、 fade gradient も出ない
- [ ] タグが 0 の書籍ではメタパネルが hidden、 シート高はコンパクト

### エッジケース

- [ ] iPhone (notch あり、 viewport 高さ 874) で 70% = 612px 程度に収まる
- [ ] 横画面 (例 viewport 高さ 390) でも内容が見える程度のサイズ
- [ ] スクロール後にメニューを × で閉じて再度開くと、 スクロール位置は上端にリセット
- [ ] backdrop タップでの閉じる動作が変わらない
- [ ] Esc キーでの閉じる動作が変わらない
- [ ] スクロール領域の touch を viewer のジェスチャー (ピンチ等) が
      横取りしない (= スクロールが正常に効く)
- [ ] iOS Safari の momentum scrolling が効く
